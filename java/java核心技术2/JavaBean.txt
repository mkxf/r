根据sun的规范，一个bean就是一个可重用的软件构件，能够在开发工具中可视化的操作。

	最简单的bena与java类没什么区别，方法采用更严格的命名。
	属性拥有get/set方法
1. JavaBean 规范没有要求bean必须有一个默认构造器，
	但大多数开发工具/框架会要求每一个bean都有，
	这样才能在没有构造参数的情况下初始化bean。

   // 实际上，jdk会自动提供一个无参的默认构造器，在没有其他
      带参构造器的情况下，如果有带参构造，就不会自动生成。
2. 属性与实例的域并不完全相同。
	概念上属性域更高一层，代表接口特征。
	实例域 属于 类的实现。
3.真正的bean要精确且冗长的多，因为：
	a  对于非专家级的程序员，bean必须便于使用
	b  同一个bean必须先在多种环境中使用。
		bean的行为与外部特征都必须可订制。
